# QUORIN — Mobile View Direction

Last updated: 2026-07-15

## Core rule

When a desktop/PC QUORIN screen or full-page reference is supplied, create the corresponding mobile experience while preserving the current QUORIN color scheme unless a different theme is explicitly requested.

Mobile is not a squeezed desktop layout.

## Experience direction

- iPhone-level perceived smoothness and polish.
- Floating navigation where appropriate.
- Selective, configurable glassmorphism rather than glass on every surface.
- Gesture-friendly commerce interactions where they improve usability.
- Large visual content and strong product imagery.
- Comfortable thumb reach and large touch targets.
- Sticky actions only when they help the current task.
- Clear hierarchy with less simultaneous information than desktop.
- Motion should communicate state and continuity, not decorate every interaction.

## Storefront mobile

Priority patterns:
- immersive hero/media,
- compact discovery navigation,
- swipe-friendly product exploration where appropriate,
- fast search and category access,
- persistent but unobtrusive cart/account access,
- product detail purchase controls optimized for one-handed use,
- no desktop mega-menu copied directly onto a phone.

## Admin mobile

Admin mobile is task-focused and responsive, not a clone of storefront mobile.

Priority tasks:
- check orders and alerts,
- search products/customers,
- inspect inventory,
- perform small operational updates,
- review dashboard signals.

Complex configuration and dense bulk workflows may remain desktop-first, but must not break on mobile.

## Validation checklist

- No horizontal overflow unless intentionally used for a carousel/table scroller.
- Touch targets are comfortably tappable.
- Bottom/floating navigation does not cover important content.
- Sticky purchase/admin actions respect safe areas.
- Text remains readable without browser zoom.
- Mobile hierarchy is intentionally redesigned when desktop density is too high.
